# Structured conversation import into Word

The browser emits a versioned neutral AST, never HTML or OOXML. The desktop Browser Imports inbox defaults to the latest assistant response plus its preceding user message, displays truncation and block statistics, and requires an exact Native Word document selection.

The desktop converts LaTeX to OMML locally and builds an immutable trusted plan. Native Word applies paragraphs, owned styles, lists, tables, and OMML inside one `UndoRecord`; an exception ends and undoes the complete record. The destination context ID is checked again immediately before dispatch. Formula numbering defaults to none.

Native Word is the first implemented structured destination. Office.js Word remains unavailable for this workflow until its bounded trusted-fragment transaction receives real-host verification. Excel, PowerPoint, WPS ET, and WPS WPP accept formulas or bounded excerpts only; full conversation import is rejected.
